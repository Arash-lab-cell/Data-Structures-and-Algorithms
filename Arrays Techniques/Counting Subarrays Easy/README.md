# Count Subarrays with Sum Less Than B

## Problem Statement

You are given:

- An integer array `A` of size `N` containing **non-negative** numbers.
- A non-negative integer `B`.

Your task is to count the number of **contiguous subarrays** whose sum is **strictly less than `B`**.

Return the total count of such subarrays.

> **Note:** It is guaranteed that there is no integer overflow during calculations.

---

## Example

### Input

```text
A = [2, 5, 6]

B = 10
```

### Output

```text
4
```

### Explanation

All possible subarrays are:

| Subarray | Sum | Valid? |
|----------|----:|:------:|
| [2] | 2 | ✅ |
| [2, 5] | 7 | ✅ |
| [2, 5, 6] | 13 | ❌ |
| [5] | 5 | ✅ |
| [5, 6] | 11 | ❌ |
| [6] | 6 | ✅ |

Total valid subarrays:

```text
4
```

---

## Another Example

### Input

```text
A = [1, 1, 1]

B = 3
```

### Output

```text
5
```

### Explanation

Valid subarrays are:

```text
[1]

[1]

[1]

[1, 1]

[1, 1]
```

The subarray:

```text
[1, 1, 1]
```

has a sum of `3`, which is **not less than** `3`.

Therefore, the answer is:

```text
5
```

---

## Approach

The solution uses a **brute-force approach**.

Every possible subarray is generated by selecting:

- A starting index.
- An ending index.

Instead of recalculating the sum for each subarray, a running sum is maintained while expanding the ending index.

Whenever the running sum is less than `B`, the count is increased.

---

## Algorithm

1. Initialize the answer as `0`.
2. Traverse every possible starting index.
3. Initialize the running sum as `0`.
4. Expand the ending index.
5. Add the current element to the running sum.
6. If:

```text
Running Sum < B
```

increment the count.

7. Continue until all subarrays have been examined.
8. Return the total count.

## Why Does This Work?

Every contiguous subarray can be uniquely identified by its:

- Starting index.
- Ending index.

The outer loop selects the starting index.

The inner loop extends the ending index while maintaining a running sum.

This running sum avoids recomputing the sum of the same elements repeatedly.

Whenever the current subarray sum is less than `B`, it contributes one to the final count.

Since every possible subarray is examined exactly once, the algorithm correctly counts all valid subarrays.

---

## Complexity Analysis

### Time Complexity

There are approximately:

```text
N × (N + 1) / 2
```

possible subarrays.

Each one is processed once using a running sum.

Overall:

```text
O(N²)
```

---

### Space Complexity

Only a few variables are used.

```text
O(1)
```

---

## Java Solution

```java
public class Solution {

    public int solve(int[] A, int B) {

        int n = A.length;

        int count = 0;

        for (int s = 0; s < n; s++) {

            int sum = 0;

            for (int e = s; e < n; e++) {

                sum += A[e];

                if (sum < B) {
                    count++;
                }
            }
        }

        return count;
    }
}
```

---

## Key Learning

- Every contiguous subarray can be generated by selecting a starting index and expanding the ending index.
- Maintaining a running sum avoids recalculating the sum for each subarray, reducing the brute-force solution from **O(N³)** to **O(N²)**.
- Since every possible subarray is examined exactly once, the algorithm guarantees the correct count.
- This solution is suitable for moderate input sizes, while larger constraints can often be optimized further using the **Sliding Window Technique**, especially because the array contains **non-negative** numbers.
- The overall time complexity is **O(N²)** with **O(1)** extra space.
